# Критерії прийняття роботи

1. Проєкт розташований у загальнодоступному репозиторії на [GitHub](https://github.com/) (можна використати альтернативу,
   таку як [GitLab](https://about.gitlab.com/) або
   [BitBucket](https://bitbucket.org/product/)).
2. Наявність коментарів та документації до коду. Присутня докладна інструкція щодо встановлення та використання
   застосунку, описана у файлі `Readme.md`.
3. Проєкт можна встановити як Python-пакет та викликати з будь-якого місця системи.
4. Коректність реалізації всіх вимог. Всі вимоги, описані вище, хоча б частково реалізовано.
5. Інтерфейс користувача реалізовано в вигляді командного рядка.
6. Інтерфейс користувача базується на текстових повідомленнях та командах, які користувач вводить з клавіатури.
7. Зручність та логічність інтерфейсу командного рядка.
8. Програма взаємодіє з користувачем в циклі, пропонуючи вибрати команду та обробляючи її, поки користувач не введе
   команду для виходу.
9. Дані коректно зберігаються на жорсткому диску та не втрачаються після перезапуску помічника.
10. Відсутність помилок у коді під час виконання застосунку.
11. Програма повинна коректно обробляти некоректне введення даних користувача без закриття програми.
12. Ефективність використання ООП, спадкування та композиції.
13. Правильна реалізація валідації для кожного поля.
14. Код повинен бути чистим, структурованим та дотримуватися стандартів [PEP 8](https://peps.python.org/pep-0008/).
